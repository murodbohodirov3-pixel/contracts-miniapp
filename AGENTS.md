# Contracts Mini App

Внутренняя система университета для учета договоров, оплат и файлов. Целевая версия работает на университетском сервере в Docker: React frontend, Node.js API и PostgreSQL. Текущий прототип хранится в `legacy/index.html` только как источник дизайна и поведения.

## Главные файлы управления проектом

- `to-do.md` — сердце текущих задач проекта и единственный оперативный список работы. Перед началом задачи нужно сверить его с фактическим состоянием проекта, записать текущую цель и ближайший шаг, а после работы кратко обновить статусы, проверки, блокеры и следующий шаг. Файл должен оставаться компактным рабочим чек-листом, а не превращаться в журнал переписки.
- `project_deploy.md` — основной план фаз разработки и развертывания целевого приложения из `legacy/index.html`. По нему определяется текущий этап, границы этапа, критерии готовности и переход к следующей фазе.
- `legacy/index.html` — источник функционального поведения, расчетов и дизайна, которые должны быть перенесены в целевую архитектуру без случайных регрессий.
- Если `to-do.md` расходится с `project_deploy.md`, сначала привести список задач в соответствие с текущей фазой из `project_deploy.md`; изменение самой фазовой стратегии требует явного решения пользователя.

## Обязательный порядок работы

- Перед изменениями сверяться с `to-do.md` и текущей фазой в `project_deploy.md`, затем сохранять функциональную совместимость с `legacy/index.html`.
- Выполнять задачи по текущей фазе, не переходить к следующей фазе до выполнения ее критериев готовности или явного решения пользователя.
- Изменения БД выполнять только новой нумерованной миграцией. Уже примененные миграции не редактировать.
- Не коммитить `.env`, пароли, ключи, выгрузки БД, загруженные договоры и резервные копии.
- После каждого завершенного обновления обновить `to-do.md`, выполнить проверки, просмотреть diff, создать понятный Git-коммит и отправить текущую ветку в GitHub.
